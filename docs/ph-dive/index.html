The Shape of Data: A Guide to Persistent Homology

Persistent homology (PH) is a popular method of topological data analysis (TDA) to find topological features in high-dimensional data. By “topological features” (structures, patterns, shapes, etc.) we mean the properties of geometric shapes and spaces that remain unchanged under continuous deformations like stretching, bending, or twisting. Examples of distinct 2D topological objects include line, branching structures (like letters K, H, F, E), and objects with holes (like D, B, A, R, 8, etc.). Sphere, torus (with one or several holes) are examples of 3D objects with distinct topological properties.

Biological data hide various topological structures. Cell states are point attractors, cell differentiation trajectory is a line with two or more branches, cell cycle or circadian rhythm are examples of circular objects. PH is especially good at finding holes and voids in data.

The procedure we’ll describe below includes the following steps.

  1. Connect the data points into a collection of “Russian-doll” graphs, each one containing the previous ones.

  2. Analyze each graph for the presence of topological features: lines, cycles, surfaces and holes.

  3. Plot the identified topological features to see which of them are transient and which are present (“persist”) in a longer sequence of the “Russian-doll” graphs.

  4. Assess the persistent features to make a conclusion about the overall topological structure of the data set.

This tutorial mostly focuses on the step 2, how to find shapes in a graph. This is where the magic is hidden and this is what algebraic topology does. As a reminder, a graph consists of vertices (nodes or “points”) and edges (lines or connections between vertices). We will work with simple 2D objects to make the analysis more tangible. In brief, the structure of a graph (its vertices and edges) is converted into a matrix, which describes how to map edges (2D structures) onto vertices (1D structures). For higher dimensions, there are more matrices that map n-dimensional structures onto a lower, (n-1) dimension. These matrices contain enough information to figure out whether points are connected into a cycle and form a boundary, and whether such a boundary has a hole in it. Linear algebra sheds the light onto a graph’s shape. This is why it is called “algebraic topology”.

One final remark before we start. This tutorial is written by an experimental biologist, not a mathematician or data scientist. It comes as a result of extensive conversations with Gemini Pro and my readings on the topic. My intention is to digest it for a person with minimal math background, like myself.

Data points get connected into a graph (a network) based on a selected distance measure. By increasing the distance threshold needed to connect two points, the data points become more and more connected. Eventually, all points are connected. We generate a series of graphs, the next graph includes the previous ones. Formally, it is called a “filtration” — a collection of sets, each one containing the previous ones. Informally, think of Russian dolls (matryoshkas).

1. The Vocabulary of Shape

To perform mathematics on a shape, we first turn a cloud of points into a rigid structure called a Simplicial Complex.

The \(n\)-Simplex: The Building Blocks

Think of a simplex as a generalized triangle. We build complex shapes out of these simple pieces:

  • 0-simplex: A single point (Vertex).

  • 1-simplex: A line segment (Edge) connecting two points.

  • 2-simplex: A filled-in triangle (Face).

  • 3-simplex: A solid tetrahedron (Void).

The Vietoris-Rips (VR) Complex

To connect data points and form a simplicial complex, we use a distance parameter \(\epsilon\) (epsilon):

  1. Draw a ball of radius \(\epsilon/2\) around every point.

  2. If two balls touch, draw a 1-simplex (edge).

  3. If three balls mutually overlap, fill in a 2-simplex (face).

The resulting structure is the Vietoris-Rips Complex. The \(n\)-skeleton refers to the collection of all simplices up to dimension \(n\) (e.g., the 1-skeleton is the graph of vertices and edges).

2. From Geometry to Algebra: Chain Groups and Boundaries

Once we have our complex, we turn it into something a computer can calculate using Chain Groups (\(C_n\)).

A Chain is a formal sum of all simplices of a certain dimension. If we have three edges, the chain space \(C_1\) is a vector space where each edge is a basis vector (an axis). Any path in the graph is just a vector in this space.

A linear algebra reminder: Basis vectors are linearly independent vectors that span a complete vector space. For example, any two vectors on a 2D plane (i.e. a 2D vector space) are linearly independent if they do not sit on the same line but rather point to different directions. Using linear combinations of these vectors, one can make any other vector on that plane. In other words, this set of vectors spans the vector space.

The Boundary Operator (\(\partial_n\))

The boundary operator is a linear map (represented as a matrix) that takes an \(n\)-dimensional object and returns its \((n-1)\)-dimensional boundary.

  • The boundary of an edge is its two endpoints.

  • The boundary of a triangle is the three edges that form its perimeter.

The Fundamental Identity: \(\partial_n \circ \partial_{n+1} = 0\).

This means the “boundary of a boundary is zero.” A boundary by definition cannot have a break or hole in it, otherwise it’s not a boundary. A closed loop doesn’t have an “end point.” A sphere does not contain an edge. This identity is the magic tool that allows us to find holes.

3. The Linear Algebra Machinery

The goal of PH is to calculate Betti Numbers (\(\beta_n\)), which count the holes:

  • \(\beta_0\): Number of connected components (islands).

  • \(\beta_1\): Number of 1D holes (circles or tunnels).

  • \(\beta_2\): Number of 2D voids (bubbles, cavities or voids).

To find these, we analyze two specific subspaces:

  1. The Kernel (\(Z_n\)): The space of Cycles. These are combinations of simplices that have a boundary of zero (e.g., a closed loop).

  2. The Image (\(B_n\)): The space of Boundaries. These are cycles that are actually just the “skin” of a higher-dimensional object (e.g., a loop filled in by a triangle).

The Quotient Space Logic

The Homology Group (\(H_n\)) is defined as the quotient space:

$$H_n = \text{Ker}(\partial_n) / \text{Im}(\partial_{n+1})$$

In a quotient space, we “re-zero” the coordinate system. We treat every vector in \(B_n\) (the filled loops) as if it were the zero vector. Any loop that cannot be “explained away” by a boundary is a true, empty hole.

4. Zooming In: A Worked Example (The 3-Point Cycle)

Let’s prove that three connected points \(\{v_0, v_1, v_2\}\) and three edges \(\{e_0, e_1, e_2\}\) form exactly one hole (\(\beta_1 = 1\)).

Step A: The Boundary Matrix (\(\partial_1\))

Rows are vertices (\(C_0\)) and columns are edges (\(C_1\)). We assign directions (\(v_0 \to v_1\), \(v_1 \to v_2\), \(v_0 \to v_2\)) to allow for algebraic cancellation.

$$\partial_1 = \begin{pmatrix} -1 & 0 & -1 \\ 1 & -1 & 0 \\ 0 & 1 & 1 \end{pmatrix}$$

Step B: Gaussian Elimination

We row-reduce this matrix to find the Rank and the Nullity.

  1. Pivot Row 1: Multiply R1 by -1.

  2. Clear R2: Subtract R1 from R2.

  3. Clear R3: Adding the resulting R2 to R3 causes Row 3 to become all zeros \([0, 0, 0]\).

The Result: * Rank = 2: It takes 2 independent edges to connect 3 vertices. \(\beta_0 = 3 - 2 = 1\).

  • Nullity = 1: We have 3 edges and a rank of 2. \(3 - 2 = 1\).

  • Cycle Vector: The kernel is spanned by \((1, 1, -1)\), which corresponds to the path \(e_0 + e_1 - e_2\).

Step C: Adding a Face (The Image)

If we fill the triangle with a face \(f_0\), we introduce the \(\partial_2\) matrix:

$$\partial_2 = \begin{pmatrix} 1 \\ 1 \\ -1 \end{pmatrix}$$

The Image of \(\partial_2\) is now exactly the same as our cycle in the Kernel. When we take the quotient, the cycle is “zeroed out.” The hole “dies” because it has been filled.

Implementation in R

The following code demonstrates how to build the boundary matrix and calculate Betti numbers manually.

# STEP 1: Define Points (Equilateral Triangle)
points <- matrix(c(0, 0, 1, 0, 0.5, 0.866), ncol = 2, byrow = TRUE)
rownames(points) <- c("v0", "v1", "v2")

# STEP 2: Distance and Connection
dist_matrix <- as.matrix(dist(points))
epsilon <- 1.1 # Threshold to connect all three points

# STEP 3: Build the Boundary Matrix (d1)
# 3 Vertices (Rows), 3 Edges (Cols)
d1 <- matrix(0, nrow = 3, ncol = 3)
colnames(d1) <- c("e01", "e12", "e02")
rownames(d1) <- c("v0", "v1", "v2")

# Orientation: Start = -1, End = 1
d1[, 1] <- c(-1,  1,  0) # e01: v0 -> v1
d1[, 2] <- c( 0, -1,  1) # e12: v1 -> v2
d1[, 3] <- c(-1,  0,  1) # e02: v0 -> v2

# STEP 4: Calculate Rank and Nullity
rank_d1 <- qr(d1)$rank
nullity_d1 <- ncol(d1) - rank_d1

# STEP 5: Betti Numbers
# Betti 0 = Vertices - Rank
betti_0 <- nrow(d1) - rank_d1
# Betti 1 = Nullity (assuming no faces are filled yet)
betti_1 <- nullity_d1

cat("Betti 0 (Connected Components):", betti_0, "\n")
cat("Betti 1 (1D Holes):", betti_1, "\n")

# STEP 6: Find the Cycle (Kernel basis) via SVD
null_space <- svd(d1)$v[, 3]
cat("Cycle Vector (Kernel):", round(null_space, 3), "\n")
